### Parch and Posey is a fictional paper-selling company, which has 50 sales representatives across four regions in the United States. Now, we'll explore a postgres database using SQL, bearing in mind the schema above, in order to answer business questions.
### This database contains following tables:
•accounts: all accounts from Parch and Posey
•orders: all orders made from 2014-12-03 to 2017-01-01
•regions: the 4 regions in the United States where Parch and Posey operates
•sales_reps: information on Parch and Posey's sales representatives
•web_events: all web event data for Parch and Posey's accounts


This script creates a SQLite database, loads csv files and performs queries to answer business questions.

### Data source: [Brazilian E-Commerce Public Dataset by Olist](https://www.kaggle.com/olistbr/brazilian-ecommerce)
